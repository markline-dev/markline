> This site is built with Markline.

## Why Markline

Point Markline at a `docs.json` and a folder of MDX, and you get a polished
documentation site — navigation, theming, search, and an interactive API
reference, all from configuration. No vendor lock-in, no hosted dependency: it
runs anywhere you can serve static files or a Node process.

## Features

- **Config-driven** navigation, theming, and branding — no code to fork.
- **First-class OpenAPI**: operation pages, schema tables, and an interactive
  **"Try it"** playground on every endpoint.
- **Static full-text search** (Pagefind) — zero search infrastructure.
- **Authoring components**: callouts, cards, steps, tabs, accordions, param fields.
- **Versioning** and **i18n** out of the box.
- **Self-hostable** as static HTML, a Node server, or on any platform.
