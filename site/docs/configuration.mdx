## Theme & branding

<ParamField path="name" type="string" required>The site name, used in the header and metadata.</ParamField>
<ParamField path="theme.logo" type="object">`{ light, dark }` image paths, or `{ text }` for a wordmark.</ParamField>
<ParamField path="theme.colors" type="object">`{ primary, primaryDark }` brand accents (hex). See [Theming](/theming) for the full palette and fonts.</ParamField>
<ParamField path="theme.appearance" type='"light" | "dark" | "system"'>Default color scheme.</ParamField>

## Topbar

<ParamField path="topbar.links" type="array">Right-aligned header links, e.g. a GitHub link.</ParamField>
<ParamField path="topbar.cta" type="object">`{ label, href }` for the primary "Get started" button.</ParamField>
<ParamField path="topbar.badge" type="string">A small label pill next to the logo (e.g. `docs · v1`).</ParamField>
<ParamField path="topbar.width" type='"full" | "contained"'>Topbar layout on docs + the API reference. `"full"` (default) spans edge-to-edge; `"contained"` centers it to the content max-width.</ParamField>
<ParamField path="topbar.homeWidth" type='"full" | "contained"'>The homepage topbar, independent of `width`. Defaults to `"contained"` so the marketing nav aligns with the hero.</ParamField>

## Navigation

Navigation is a list of tabs, each with grouped pages. A tab can instead set
`openapi: true` to generate its sidebar from your OpenAPI spec.

```jsonc
"navigation": {
  "tabs": [
    { "id": "docs", "label": "Documentation", "href": "/", "match": ["__default__"],
      "groups": [{ "group": "Start", "pages": [{ "href": "/", "label": "Intro" }] }] },
    { "id": "api", "label": "API", "href": "/api-reference", "match": ["/api-reference"], "openapi": true }
  ]
}
```

## API & playground

<ParamField path="api.baseUrl" type="string">Base URL the playground sends requests to (overrides the spec's servers).</ParamField>
<ParamField path="api.playground.mode" type='"full" | "inline" | "explorer" | "off"'>The "Try it" experience. Default `"full"`.</ParamField>
<ParamField path="api.playground.proxy" type='"auto" | "always" | "never"'>How the playground reaches your API. `never` is required for static export.</ParamField>

See [API reference](/openapi) for the full guide — generating the reference, the live explorer, and per-resource/per-operation MDX overlays.

## Integrations

<ParamField path="editUrl" type="string">Base URL for "Edit this page" links; the page path is appended.</ParamField>
<ParamField path="analytics" type="object">`plausible`, `googleAnalytics`, or `posthog`.</ParamField>
<ParamField path="feedback.endpoint" type="string">POST target for the "Was this helpful?" widget.</ParamField>

## Versions & i18n

<ParamField path="versions" type="array">Documentation versions; the first is the default (unprefixed).</ParamField>
<ParamField path="i18n.locales" type="array">Localizations; the first locale is the default. Same mechanism as versions.</ParamField>

<Note>A project uses versions **or** locales — both share the URL-prefix mechanism.</Note>
