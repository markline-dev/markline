## Install & scaffold

<Steps>
  <Step title="Scaffold a project">
    ```bash
    npm create @markline@latest my-docs
    ```
    This creates a `markline.json`, sample MDX pages, an example OpenAPI spec, and
    ready-to-use deploy configs.
  </Step>
  <Step title="Install & run">
    ```bash
    cd my-docs
    npm install
    npm run dev
    ```
  </Step>
</Steps>

<Note>Already have a project? `npm install @markline/markline` then `npx markline init`.</Note>

## Project structure

Your repository holds only content and configuration:

```text
my-docs/
├── markline.json     # navigation, theme, branding, API settings
├── docs/             # your pages (*.mdx)
│   ├── index.mdx
│   └── quickstart.mdx
└── api/
    └── openapi.json   # rendered as an interactive API reference
```

<Note>The OpenAPI file is optional — Markline works as a docs-only site too.</Note>

## Develop

Edit files under `docs/` and update `markline.json` to change navigation, branding,
and the API base URL. When you're ready, see [Deployment](/deployment).
